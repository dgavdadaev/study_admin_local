# Цель работы

## Цель

Изучение принципов передачи данных в локальной сети  
с использованием Cisco Packet Tracer.

# Ход выполнения

## Создание топологии с концентратором

- Размещён Hub-PT и 4 ПК  
- Устройства соединены прямыми кабелями  
- Настроены IP-адреса:
  - 192.168.1.11–192.168.1.14

![Топология с концентратором](Screenshot_1.png){ width=70% }

## Передача пакетов

- Создан PDU: PC0 → PC2  
- Наблюдение ARP и ICMP  
- Отслежено движение пакетов

![Передача ARP и ICMP](Screenshot_2.png){ width=70% }

## Особенности Hub

- Работает на физическом уровне  
- Передаёт сигнал на все порты  
- MAC-адреса не анализируются  

![Модель OSI в Hub](Screenshot_3.png){ width=80% }

## Коллизии

- Созданы два одновременных обмена  
- Возникло столкновение сигналов  
- Повреждение кадров и повторная передача  

![Коллизии в сети с Hub](Screenshot_5.png){ width=70% }

## Новая топология с коммутатором

- Добавлен коммутатор Cisco 2950  
- Подключены 4 ПК  
- Назначены IP:
  - 192.168.1.21–192.168.1.24  

![Топология с коммутатором](Screenshot_8.png){ width=70% }

## Передача данных

- PDU: PC4 → PC6  
- Наблюдение ARP и ICMP  
- Коммутатор запоминает MAC-адреса  

![Передача через switch](Screenshot_9.png){ width=70% }

## Отличия от концентратора

- Кадры отправляются только на нужный порт  
- Уменьшение сетевой нагрузки  
- Отсутствие коллизий  

![Одновременная передача без коллизий](Screenshot_11.png){ width=70% }

## Соединение Hub и Switch

- Hub соединён со switch  
- Выполнен обмен PC0 ↔ PC4  

![Объединённая сеть](Screenshot_12.png){ width=75% }

## Поведение сети

- Сначала возникает коллизия  
- Затем повторная передача  
- Пакеты успешно достигают цели  

![Коллизия и успешная доставка](Screenshot_13.png){ width=75% }

## Протокол STP

- Получены служебные STP-пакеты  
- Формат Ethernet 802.3  
- Используется групповой MAC-адрес  

![Структура STP](Screenshot_15.png){ width=80% }

## Настройка Router

- Подключён к коммутатору  
- Назначен IP:
  - 192.168.1.254 /24  
- Интерфейс активирован  

![Подключение маршрутизатора](Screenshot_16.png){ width=75% }

## Передача PC → Router

- Наблюдаются ARP и ICMP  
- Дополнительно фиксируются STP и CDP  

![Передача пакетов к маршрутизатору](Screenshot_17.png){ width=75% }

## Протокол CDP

- Кадры IEEE 802.3 + SNAP  
- MAC назначения:
  - 01:00:0C:CC:CC:CC  
- TLV-структура данных  

![Структура CDP](Screenshot_18.png){ width=85% }

В ходе работы:

- Сравнена работа Hub и Switch

- Рассмотрена структура Ethernet-кадров
