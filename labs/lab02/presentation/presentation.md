# Цель работы

## Цель

Получить основные навыки по начальному конфигурированию  
сетевого оборудования Cisco в Packet Tracer.

# Ход выполнения

## Создание топологии

- Размещены:
  - маршрутизатор 2811  
  - коммутатор 2960-24TT  
  - два ПК  
- Один ПК подключён к маршрутизатору  
- Второй ПК подключён к коммутатору  
- Использован прямой медный кабель  

![Топология сети](Screenshot_1.png){ width=70% }

## Настройка PC1 (через коммутатор)

- IP-адрес: 192.168.2.10  
- Маска: 255.255.255.0  
- Шлюз: 192.168.2.1  

![Настройка IP PC1](Screenshot_2.png){ width=70% }

## Настройка PC0 (через маршрутизатор)

- IP-адрес: 192.168.1.1  
- Маска: 255.255.255.0  
- Шлюз: 192.168.1.254  

![Настройка IP PC0](Screenshot_6.png){ width=70% }

## Настройка коммутатора

- Задано имя устройства  
- Создан VLAN 2  
- Назначен IP: 192.168.2.1  
- Настроен порт access VLAN 2  
- Указан шлюз: 192.168.2.254  

![Конфигурация коммутатора](Screenshot_3.png){ width=80% }

## Настройка удалённого доступа (Switch)

- Пароли на console и VTY  
- enable secret  
- user admin  
- Генерация RSA-ключей  
- Включён SSH  

![SSH на коммутаторе](Screenshot_4.png){ width=80% }

## Настройка маршрутизатора

- Интерфейс Fa0/0:  
  - 192.168.1.254 /24  
- Интерфейс активирован (no shutdown)

![Конфигурация маршрутизатора](Screenshot_7.png){ width=80% }

## Настройка удалённого доступа (Router)

- Пароли console и VTY  
- enable secret  
- user admin  
- domain-name  
- RSA-ключи  
- Разрешён SSH  

![SSH на маршрутизаторе](Screenshot_8.png){ width=80% }

## Проверка соединения (Switch)

- Выполнен ping 192.168.2.1  
- Успешные ответы  
- Подключение:
  - telnet  
  - ssh  

![Проверка доступа к коммутатору](Screenshot_5.png){ width=80% }

## Проверка соединения (Router)

- Выполнен ping 192.168.1.254  
- Ответ без потерь  
- Успешное подключение по SSH  

![Проверка доступа к маршрутизатору](Screenshot_9.png){ width=80% }
